# TS Documentation

Use this skill when writing or updating technical docs for TS codebases.

## Checklist
- Document intent and contracts, not obvious syntax.
- Include minimal runnable examples where helpful.
- Keep docs close to source of truth.
- Explain extension points and invariants.
- Capture assumptions and operational constraints.
- Keep wording concise and implementation-aligned.
