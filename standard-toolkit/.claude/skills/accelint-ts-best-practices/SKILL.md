# TypeScript Best Practices

Use this skill for TypeScript implementation and review.

## Checklist
- Enable strict typing and avoid `any` in app code.
- Model domain contracts with explicit interfaces/types.
- Use runtime validation at I/O boundaries (e.g., zod).
- Keep shared types in dedicated packages.
- Encode impossible states through discriminated unions.
- Favor small, typed utility functions over broad helpers.
