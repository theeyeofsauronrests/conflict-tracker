# TypeScript Testing

Use this skill for TS business-logic and API-route tests.

## Checklist
- Cover success and failure branches.
- Keep tests deterministic and side-effect isolated.
- Validate typed contracts at boundaries.
- Prefer table-driven tests for branch-heavy logic.
- Keep mocks local and explicit.
- Fail fast with clear assertion messages.
